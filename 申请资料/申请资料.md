# FeatureStore — ML 特征工程流水线

## 基本信息

- **项目名**：FeatureStore
- **GitHub**：https://github.com/dongjieliang8-blip/featurestore
- **技术栈**：Python + Claude Code + MiMo API + pandas + scikit-learn
- **Agent 数量**：4
- **单次 Token 消耗**：200-400 万

## 04 字段文本

我构建了一个名为 **FeatureStore** 的多 Agent 协作 ML 特征工程流水线系统，基于 Claude Code 开发、MiMo API 驱动。该项目解决的核心痛点是：机器学习项目中特征工程耗时占建模流程 60-80%，特征质量参差不齐、特征复用率低、线上线下特征不一致导致模型效果波动。FeatureStore 通过 4 个角色分工明确的 AI Agent 实现特征提取→特征验证→特征注册→特征服务的完整自动化闭环。

核心逻辑流采用长链推理架构：第一层 Feature Extractor Agent 对原始数据进行智能特征提取，支持数值特征（统计聚合、分箱、交叉）、类别特征（编码、频率、目标编码）、时序特征（滑窗、趋势、周期）和文本特征（TF-IDF、嵌入），输出结构化特征定义和衍生逻辑；第二层 Feature Validator Agent 对提取的特征进行质量验证，检查完整性（缺失率）、分布（偏度、峰度）、相关性（多重共线性）和业务逻辑合理性；第三层 Feature Registry Agent 管理特征元数据，包括版本控制、血缘关系、状态管理和标签分类，支持特征复用和依赖追踪；第四层 Feature Serving Agent 设计特征服务方案，支持离线批处理、在线实时、流式计算等多种服务模式，输出完整的特征工程流水线配置。四个 Agent 间通信全部采用结构化 JSON，形成可追溯、可审计的工程链路。

项目使用 Python 构建，CLI 基于 Click + Rich 实现终端可视化。单次完整流水线运行消耗约 200-400 万 Token。目前已在个人实验室环境中投入使用，特征工程效率提升约 70%。

项目地址：https://github.com/dongjieliang8-blip/featurestore
